If you want to add Microconfig-core as dependency to your Java project this Maven artifact.

See documentation how to run config build via CLI.

What Microconfig can do

#Includes

You can extract common part for different microservices to a dedicated component and then reuse it via #include

${Place@holders}

Instead of duplicating the value of some properties you can use a reference to this value.

#{'Expression' + 'language'}

If #include and ${placeholders} is not enough you can use powerful expression language for dynamic properties

Environment specific configs

For each microservice you can write common configuration for all environment and specify environment specific properties (add/remove/override).

Templates

Microconfig allows you to keep configuration files for any libraries in their specific format and resolve placeholders inside them.

Config diff

Microconfig can detect added/removed/changed properties for each service and store this information after every config build.
