Microservice configuration
made easy

Microservice architecture implies a lot of services with similar configuration.
Each service has an application config, deployment config, a log template and so on.
Multiply that by 50 services with 5 different environments and you have a mess.

How Microconfig can help

Simple Configuration Layout

You choose your configuration layout. You can group and organize your config the way it makes sense to you.

Reuse Common Parts

You can extract common parts of configuration to a dedicated component and reuse it with #include.

Reference Values

You can reference a value from current or another service with ${placeholder}.

Dynamic Values

You can use #{ 'expression' + 'language' } to generate your values. It supports math operations and much more.

Environment Specific Config

You can have specific values for each environment you need. They will be merged with your base configuration.

Static Files Templating

You can use static templates and populate them with data unique for each service.

Config Diff

You can generate git style diff and compare different versions of your config.

IDE Plugin

You can use JetBrains IDE Microconfig plugin to navigate, preview and generate config without leaving your IDE.

Just to show the problem with microservice configuration more explicitly. Below are typical configs that we have to support nowadays. 2 different microservices with their dev and prod configs.

payment-backend log config

                                        </encoder>
                                      </appender>
                                    </configuration>
